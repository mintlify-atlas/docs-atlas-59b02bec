title: "ISO Settings"
description: "Customise the filename template used when creating macOS Bootable Installer Disk Images (ISOs)."
---

Access these settings from **Mist &gt; Settings &gt; ISOs**.

## Overview

ISOs are **Bootable macOS Installer Disk Images** that can be used with virtualization software such as [Parallels Desktop](https://www.parallels.com/products/desktop/), [UTM](https://mac.getutm.app), [VMware Fusion](https://www.vmware.com/products/fusion.html), and [VirtualBox](https://www.virtualbox.org).

<Note>
  ISOs are unavailable for building **macOS Catalina 10.15 and older** on [Apple Silicon Macs](https://support.apple.com/en-us/HT211814). The ISO option is automatically hidden for those combinations.
</Note>

---

## ISO Filename

The **ISO filename** field controls the name given to each generated `.iso` file. It supports dynamic template variables that are substituted with real values at build time.

### Template variables

| Variable | Replaced with | Example |
|---|---|---|
| `%NAME%` | The macOS release name | `macOS Tahoe` |
| `%VERSION%` | The macOS version number | `26.0` |
| `%BUILD%` | The build identifier | `25A354` |

### Default template

```
Install %NAME% %VERSION%_%BUILD%.iso
```

### Example output

With the default template and a macOS Tahoe 26.0 installer:

```
Install macOS Tahoe 26.0_25A354.iso
```

The preview below the filename field updates in real time as you type.

---

## Reset to Defaults

Clicking **Reset to Defaults** restores the ISO filename to:

```
Install %NAME% %VERSION%_%BUILD%.iso
```
